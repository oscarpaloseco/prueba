# AGENTS.md — prueba design system (v0.1.1)

> **AI entry guide.** Read this before using any component or token from
> this design-system package.

## What this is

This package is the exported design system for **prueba**, generated by the
DS Foundations Engine. It contains typed React components and a CSS token
file (`tokens.css`).

## Golden rules

1. **Prefer these components.** Don't invent new UI primitives — pick from
   the component index below.
2. **Reference tokens as `var(--ds-*)`.** Never hard-code hex values.
   Import `tokens.css` once at your app root.
3. **Respect token families:**
   - `surface.*` → background fills
   - `content.*` / `text.*` → text / foreground colour
   - `border.*` → stroke / border colours
   - `icon.*` → glyph / SVG fill colour
4. **Don't invent styles.** Only override tokens; don't write new CSS that
   contradicts the design system.
5. **Read the component `.md` before instantiating.** Each doc lists the
   available props, when to use, and dos/don'ts.

## Importing components

```tsx
// Once in your app root:
import "@prueba/design-system/tokens.css";

// Then anywhere:
import { Button, Input } from "@prueba/design-system";
```
